 AI Scaffolding Prompt: ALGO_V_02 
 Feature Expansion 
 Objective:  Expand the existing Algorithm Library platform  by integrating a "Race Mode" 
 benchmarking view, an Edge-Case Data Injector, active code synchronization in the UI, and 
 global execution speed controls. Maintain the established dark, precision-focused styling and 
 the decoupled generator-function execution architecture. 
 1. Global Execution Controls (Speed Adjuster) 
 We need a way to control the interval at which the Zustand store iterates the active generator 
 function. 
 ●  Zustand Store Update:  Add a playbackSpeed state (default:  1x, ranging from 0.25x to 4x 
 or "Max"). Update the play() action's `setInterval` duration to be dynamically tied to this 
 playbackSpeed value. 
 ●  UI Implementation:  Add a speed control dropdown or  slider adjacent to the existing 
 playback controls (Rewind, Play/Pause, Forward) in the VisualizerController. 
 2. Edge-Case Data Injector (Custom Input Module) 
 Enhance the sidebar or visualizer header to allow users to override the default randomized 
 array with specific data permutations. 
 ●  UI Implementation:  Create a new DataInjector component  containing quick-action 
 buttons: "Sorted", "Reverse Sorted", "Nearly Sorted", "Many Duplicates". Include an input 
 field for custom array sizes (e.g., $N=10$, $N=100$). 
 ●  Zustand Store Update:  Ensure the store has an action  to reset the generator function 
 and the UI state completely when a new dataset is injected, initializing the visualization 
 canvas with the new data. 
 3. Code Pane Enhancements (Active Sync) 
 The code pane must intelligently respond to the algorithm's current state to ensure the user 
 always sees the active execution line. 
 ●  Highlighting Logic:  Ensure the Prism.js (or equivalent)  renderer applies a distinct active 
 background class (e.g., a low-opacity variant of the primary neon green, like 
 bg-green-500/20) strictly to the <li> or line matching the activeCodeLine yielded by the 
 generator. 
 ●  Auto-Scroll Implementation:  Use a React useRef attached  to the highlighted line 
 element. Implement a useEffect that triggers scrollIntoView({ behavior: 'smooth', block:  'center' }) whenever the activeCodeLine state changes, ensuring deep loops do not drift 
 off-screen. 
 4. "Race Mode" (Algorithm Benchmarking V iew) 
 Create a new dedicated route and layout for side-by-side performance benchmarking. 
 ●  New Route:  /benchmark or /race. 
 ●  Layout Requirements:  Implement a vertically stacked  split-pane (top and bottom). The 
 top pane hosts VisualizerInstance A (e.g., Bubble Sort) and the bottom pane hosts 
 VisualizerInstance B (e.g., Quick Sort). Both must share the exact same generated 
 dataset array. 
 ●  State Management:  The Zustand store needs to handle  dual execution. The global 'Play' 
 button must trigger  both  generators simultaneously. 
 ●  Metrics Display:  Add  a live "Operations/Tick" counter alongside the existing 
 "Comparisons" and "Swaps" counters to visually emphasize the speed difference between 
 $O(N^2)$ and $O(N \log N)$ complexity when rendering at the same playback speed.
